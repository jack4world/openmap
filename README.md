# OpenStreetMap (OSM) 学习与项目计划

本项目旨在通过徐汇区的办公楼与星巴克分布，探索复杂的地理空间分析（GIS）场景，并建立一套完整的开源 GIS 技术栈学习路径。

---

## 📍 当前分析场景：复杂地理分析 (Spatial Analysis)
**案例**：寻找上海徐汇区方圆 1 公里内没有星巴克但有 5 个以上写字楼的坐标。

### 已实现功能
- **数据获取**：通过 Overpass API 抓取徐汇区 `building=office` 和 `amenity=cafe` 数据。
- **空间计算**：进行写字楼密集度与咖啡馆距离的关联分析。
- **可视化**：使用 **Kepler.gl** 生成交互式 3D 关联地图 (`kepler_map.html`)。

## 🛠 开发者选型建议与业务应用场景

针对不同的项目需求，建议开发者选择最合适的工具栈：

### 1. 简单展示场景 (Simple Display)
*   **选型建议**：Leaflet + MapTiler/Stadia Maps (底图服务商)
*   **典型应用**：企业官网“联系我们”、活动落地页、个人博客旅行打卡、轻量级 H5 门店展示。
*   **优势**：开发成本极低，几行 JS 即可上线，无需维护服务器，加载极快。

### 2. 复杂地理分析场景 (Spatial Analysis) —— 本项目所属
*   **选型建议**：**PostGIS + Python (GeoPandas) + Overpass API**
*   **典型应用**：商业选址（如本项目）、城市规划研究（绿地率/路网密度）、房产配套设施距离计算、保险风险评估。
*   **优势**：强大的 SQL 空间查询（ST_Intersects, ST_Distance），Python 科学计算生态支持复杂算法。

### 3. 高性能移动端/导航场景 (Mobile & Routing)
*   **选型建议**：MapLibre SDK + Valhalla/OSRM + 矢量切片
*   **典型应用**：外卖/快递配送、打车应用（ETA 计算）、户外运动 App（轨迹记录/高度分析）、车载导航。
*   **优势**：矢量切片缩放平滑且支持动态样式，C++ 引擎支持每秒万次路径请求。

### 4. 完全自主掌控场景 (Enterprise/Self-Hosted)
*   **选型建议**：自建 TileServer GL + 自建 Nominatim + Planet.osm
*   **典型应用**：超高并发商业应用（节省账单）、内网/涉密项目、完全定制化地图风格、离线地图包。
*   **优势**：数据隐私安全，长期运行成本极低且拥有完全的控制权。

## 🗺 选型决策对照表

| 需求优先级 | 建议方案 | 学习曲线 | 成本 | 代表场景 |
| :--- | :--- | :--- | :--- | :--- |
| **快速上线** | Leaflet + 第三方底图 | 低 | 低 (免费额度) | 官网、活动页 |
| **数据挖掘** | PostGIS + Python | 中 | 中 (需数据库) | 商业分析、科研 |

## 🏔 户外爱好者与 3D 深度应用

OSM 的强大在于其开放性，在户外和 3D 场景中表现卓越：

### 1. 户外核心场景
*   **徒步登山**：利用 `sac_scale`（难度）和 `trail_visibility` 获取专业路径信息。建议使用 **OpenTopoMap** 样式。
*   **骑行运动**：通过 `surface`（路面材质）决定车型，利用 `slope`（坡度）规划体力。
*   **离线地图**：户外爱好者刚需，建议使用 **Tilemaker** 预处理矢量切片。

### 2. 3D 地图展示与分析
*   **2.5D 挤压**：利用 `height` 和 `building:levels` 标签让城市建筑“立起来”（MapLibre fill-extrusion）。
*   **地形 3D**：结合外部 DEM（高程数据）实现山脉起伏，支持航拍视角巡航。
*   **阴影分析**：根据地理坐标和建筑高度计算滑雪场或徒步路径的动态阴影遮挡。

- `fetch_data.py`: 数据抓取脚本。
- `generate_kepler_map.py`: 生成 Kepler.gl 可视化网页。
- `xuhui_roads.*`: 徐汇区道路网路数据。
- `kepler_map.html`: 最终生成的交互式地图文件。
