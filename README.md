# OpenStreetMap (OSM) 学习与项目计划

本项目以"上海徐汇区星巴克选址真空区"为实战案例，探索基于 OpenStreetMap 的复杂地理空间分析（GIS）技术栈。

---

## 📍 当前分析场景：复杂地理分析 (Spatial Analysis)

### 徐汇区办公楼与星巴克选址真空区分析 (Vacuum Analysis)

**案例目标**：寻找上海徐汇区方圆 1 公里内没有星巴克但写字楼密集的坐标，发掘潜在商业选址机会。

**数据规模**：通过 Overpass API 抓取徐汇区全量数据 —— **157 个办公点**、**53 家星巴克**。

### 🔍 核心发现

对每个办公点进行 1 公里半径缓冲区分析后，得出以下分布：

| 覆盖等级 | 办公点数量 | 占比 | 说明 |
| :--- | :--- | :--- | :--- |
| **零覆盖** (0 家星巴克) | 33 | 21% | 选址真空区，高潜力 |
| **低覆盖** (1-3 家) | 71 | 45% | 配套不足，有补位空间 |
| **高覆盖** (10+ 家) | 18 | 11% | 商业成熟区，趋于饱和 |

#### 🏗 顶级真空区（零星巴克覆盖的写字楼集群）
- **华泾/龙吴路片区** (31.11, 121.449)：3 个办公点聚集，方圆 1 公里内 **0 家星巴克**。
- **长桥/康健片区** (31.139, 121.449)：2 个办公点无覆盖。
- **漕河泾科汇大厦周边** (31.18, 121.397)：科技园区办公密集，咖啡配套空白。

#### ☕ 最高饱和区（对照组）
- **徐家汇核心区** (禾念/气象局/港汇周边)：1 公里内星巴克多达 **15-16 家**，已高度饱和。

## 🛠 开发者选型建议与业务应用场景

| 需求优先级 | 建议方案 | 学习曲线 | 代表场景 |
| :--- | :--- | :--- | :--- |
| **快速上线** | Leaflet + MapTiler/Stadia Maps | 低 | 官网地图、活动落地页 |
| **数据挖掘** | **PostGIS + Python + Overpass API** ⬅ 本项目 | 中 | 商业选址、城市规划 |
| **极致交互** | MapLibre SDK + Valhalla/OSRM | 高 | 导航、外卖、运动App |
| **大规模/内网** | 自建 TileServer GL + Nominatim | 极高 | 涉密项目、高频API |

## 📁 项目文件说明

### 数据获取
| 文件 | 说明 |
| :--- | :--- |
| `fetch_data.py` | 通过 Overpass API 抓取徐汇区办公楼 + 星巴克 POI 数据 |
| `fetch_roads.py` / `fetch_roads_xml.py` | 抓取徐汇区道路网络数据 |
| `xuhui_data.json` / `.geojson` | 办公楼与星巴克原始数据 |
| `xuhui_roads.json` / `.geojson` / `.osm` | 道路网络数据（多格式） |

### 数据处理与转换
| 文件 | 说明 |
| :--- | :--- |
| `convert_roads.py` | 道路数据格式转换 |
| `convert_to_geojson.py` | JSON → GeoJSON 转换 |
| `json_to_osm.py` | JSON → OSM XML 转换 |
| `query.sql` / `routing_query.sql` / `export_csv.sql` | PostGIS 空间查询与路径分析 SQL |

### 分析与可视化
| 文件 | 说明 |
| :--- | :--- |
| `generate_kepler_map.py` | 生成 Kepler.gl 交互式真空区分析地图 |
| `kepler_vacuum_zones.csv` | 157 个办公点的星巴克覆盖数据（核心分析结果） |
| `office_density_vs_starbucks.csv` | 网格聚类密度分析数据 |
| `kepler_vacuum_map.html` | 交互式真空区可视化（本地打开） |

---

> **入门建议**：初学者从 Leaflet 展示入手；需要处理业务数据时，尽快学习 PostGIS + Python 组合。OSM 的核心价值在于其数据的开放性 —— 任何人都可以基于同一份数据构建完全不同的分析视角。
